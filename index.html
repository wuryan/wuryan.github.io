Java多线程实战（1）

Posted on 2017-01-01

Java多线程实战（1）

标签（空格分隔）： java 多线程 并发

线程是什么？

从底层来说线程其实就是一块存储CPU指令的内存空间，CPU为每一个线程分配时间片，每一个时间片周期内，CPU会按照线程的指令顺序执行，如果一个时间片周期内CPU没有全部执行完线程中的指令，则CPU会记录执行到哪个指令，然后切换到下个时间片，以此反复。那么就可以把线程比喻成要做事的人，人做事都是顺序执行的。

搞清楚并发和并行

并发和并行是常常被混淆的概念。我刚学习编程的时候也是将这两个概念混淆了，导致和前辈们聊天时闹了笑话。现在就来理清楚并发和并行的概念有和不同。

  • 并发：并发其实就是事件同事发生了，但是处理起来是一步一步的执行，在某一瞬间是只能执行一件事。在CPU上的体现就是单个CPU的时间片轮转。只不过CPU时间片轮转切换的速度非常快，对于我们来说几乎是一瞬间的事，所以我们感觉好像是同时进行。
  • 并行：并行和并发一样，事件也是同时发生，但是处理起来却是并排的执行，就是在某一瞬间是执行了多件事。在CPU的体现就是在两个CPU上分别执行。

所以并行和并发是有根本区别的。但是我们看起来似乎是一样，这也是导致我们经常混淆这两个概念。

同步和异步

  • 同步：
    比如说领导把任务交给小张来做，小张就是一个名为“小张”的线程。若果小张一个人在做这个任务，不论这个任务有多难要做多久，小张都要一个人完成，这个就是同步。如果这个时候领导又分配给小张其他的任务，那么也必须等小张将当前的任务做完。
  • 异步：
    如果小张将这个任务交给另外一个人“小王”来做，那么小王就开启了一个叫做“小王”的线程来做这个任务，然后小张就可能忙其他任务去了，那么这个就是异步。如果这个时候领导又分配给小张其他任务，小张就可以做新任务。

临界区

临界区其实就是一种共享数据区或者说是一个公共资源区。比如说公共厕所就是一个共享资源，每个人都可以进去，但是这个资源是有限的，当里面有人时，就会把门锁住，外面的人想进去就必须等待里面的人出来。

阻塞和非阻塞

阻塞和非阻塞通常用来形容多个线程同时访问临界区时的相互影响。

  • 阻塞
    小张接到领导分配的任务，需要用到打印机（临界区，共享资源），当他去使用打印机的时候，发现打印机已经被小王使用了，小张这时候就在打印机前面等待小张使用完毕再使用，这就是阻塞。
  • 非阻塞
    和阻塞同样的场景，小张这时候回去做这个任务的其他部分，过一会再来使用打印机，这就是非阻塞。

同步和异步、阻塞和非阻塞有何异同？

“同步和异步”主要侧重于单个线程的执行，而“阻塞和非阻塞”主要侧重于多个线程间的共享资源访问。
